# Post-generation consistency (reference)

После генерации или крупного редактирования пройди **консистентность** по репозиторию.

## Имена и контракты

- [ ] Одинаковые строки сценариев/API на frontend и backend (или общий источник).
- [ ] Переменные окружения: пример в README совпадает с `docker-compose` и кодом.

## Prisma и БД

- [ ] Схема Prisma согласована с миграциями; нет «ручных» расхождений без миграции.

## Observability

- [ ] Запросы в Grafana/Prometheus используют актуальные имена метрик и лейблов.
- [ ] Promtail/Loki пути и labels не противоречат тому, что пишет приложение.

## Документация

- [ ] README не обещает сервисов, которых нет в compose, и наоборот.

## Финал

- [ ] Нет мёртвого кода, оставшегося от промежуточных попыток (временные debug-логи, закомментированные блоки без причины).
