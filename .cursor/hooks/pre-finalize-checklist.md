# Pre-finalize checklist (reference)

Используй как ручной или ментальный чеклист **перед** тем как считать ответ/изменения финальными. Для автоматизации см. `hooks.json` и command hooks в `.cursor/hooks/`.

## Обязательно

- [ ] Запрос пользователя закрыт полностью (не только часть).
- [ ] Соблюдены guardrails стека (`.cursor/rules/stack-guardrails.md`).
- [ ] Нет лишних файлов и массового рефакторинга вне задачи.
- [ ] Сборка/типы/линт для затронутых областей не ухудшены.

## Если менялся код приложения

- [ ] Definition of done (`.cursor/rules/definition-of-done.md`) — релевантные пункты отмечены по смыслу.

## Если менялись сценарии или backend

- [ ] Логи и метрики по-прежнему осмысленны; имена сценариев согласованы с README.

## Перед сообщением пользователю

- [ ] Указано, как проверить результат (команда или шаги).
