# Definition of done

Считай задачу выполненной только если выполнено по смыслу запроса **и** проверено ниже.

## Функциональность

- [ ] Поведение соответствует описанию задачи; граничные случаи не сломаны без явного согласования.
- [ ] Публичные контракты (API, env, compose) обновлены, если изменились.

## Сборка и типы

- [ ] Проект собирается / типы проходят там, где это принято в репо (`npm`/`pnpm`/`yarn` + Prisma generate при необходимости).
- [ ] Нет намеренно подавленных ошибок линтера в изменённых файлах.

## Observability (если затронут backend или сценарии)

- [ ] Логи остаются структурированными; при новых путях — понятные поля (request id, scenario, status).
- [ ] Метрики/лейблы согласованы с существующими dashboard’ами или добавлены с пояснением.
- [ ] Для сценариев: запуск по-прежнему даёт HTTP-ответ, запись в БД, лог, обновление метрик (и ошибку в Sentry при включённом Sentry).

## Документация

- [ ] README или внутренние заметки обновлены, если менялся способ запуска, переменные окружения или список сценариев.

## Передача пользователю

- [ ] Кратко перечислено, что изменилось и как проверить (команда или шаги в UI).
