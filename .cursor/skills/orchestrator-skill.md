# Orchestrator

## Принципы

1. **Порядок**: схема данных и Prisma → backend (NestJS) → контракт API → frontend → compose/observability → README.
2. **Один источник правды**: типы/DTO согласованы между слоями; не дублировать магические строки сценариев — вынести в общий enum/константы там, где уже принято.
3. **Миграции**: одна логическая миграция на изменение схемы; не смешивать с рефакторингом UI.

## Шаги

1. Прочитать README и существующие сценарии/эндпоинты.
2. Составить краткий план файлов (какие трогаем).
3. Выполнить изменения слой за слоем; после Prisma — `generate` и проверка типов.
4. Поднять stack локально при необходимости и прогнать один сценарий end-to-end.
5. Обновить definition-of-done: сборка, логи, метрики, документация.

## Когда остановиться и спросить

- Смена БД или ORM, отказ от Docker Compose, продакшен-секреты — только после явного согласования пользователя.
