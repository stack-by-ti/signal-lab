# Add scenario

## Цель

Добавить предсказуемый сценарий, который проходит через API, БД, логи и метрики (и при необходимости Sentry).

## Шаги

1. **Найти существующие сценарии** в backend (контроллеры/сервисы, enum или конфиг) и во frontend, если есть выбор сценария.
2. **Имя сценария**: `snake_case`, одно слово или несколько через `_`, без пробелов; краткое описание для README.
3. **Поведение**:
   - Успех: стабильный ответ и запись в БД.
   - `slow_request`: искусственная задержка с разумным таймаутом и документацией.
   - `system_error` или аналог: контролируемая ошибка с корректным HTTP-кодом и логом уровня error.
4. **Observability**: не убирать существующие spans/поля логов; добавить поле `scenario` (или принятое в проекте имя), если его ещё нет.
5. **Документация**: строка в таблице сценариев в README (как в проекте сейчас).
6. **Проверка**: один ручной или автоматический прогон через compose/API по инструкции в README.

## Анти-паттерны

- Скрытые сценарии без README.
- Ошибки без структурированного лога и без понятного кода ответа.
